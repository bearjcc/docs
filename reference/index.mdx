Standards for writing and maintaining this Mintlify site.

## Pages

- [MDX components and patterns](/reference/mdx-tips) — callouts, steps, tabs, code, diagrams, images
- [Authoring conventions](/reference/authoring-conventions) — frontmatter, links, navigation, DriveWorks terminology, custom function docs

## Repo layout

| Path           | Purpose                                     |
| -------------- | ------------------------------------------- |
| `docs.json`    | Site config, theme, tabs, navigation        |
| `**/*.mdx`     | Documentation pages                         |
| `snippets/`    | Reusable React components imported into MDX |
| `scripts/`     | Generators (e.g. architecture layer tables) |
| `images/wiki/` | Static assets (migrated Wiki media, drawio) |

## Cursor rules

These rules auto-attach when editing `.mdx` in this repo (`.cursor/rules/`):

| Rule                          | Covers                                                               |
| ----------------------------- | -------------------------------------------------------------------- |
| `mintlify-mdx`                | Components, snippets, navigation, MDX syntax                         |
| `Docs-Mintlify-Diataxis`      | One page type per page (tutorial / how-to / reference / explanation) |
| `Docs-Mintlify-Writing-Style` | Inverted pyramid, concise "why", active voice, banned filler words   |
| `Docs-Mintlify-Interlinking`  | Link once with intent; single source of truth                        |
| `Docs-Mintlify-IA-Adoption`   | Use / Maintain / Internal lanes, task-based entry, maintenance       |

## MCP

Mintlify publishes a [Cursor writing guide](https://www.mintlify.com/docs/guides/cursor) and hosts a search MCP at `https://mintlify.com/docs/mcp` for component reference while authoring.
