DriveWorks access is layered: login, DriveApp by region or team, Project run/edit rights, and catalog visibility. Set these in **Security Settings** (Stage 1) and the **DriveApp Admin** dashboard (Stage 7), and revalidate after any Group copy or restore.

## Permission layers

1. DriveWorks login access.
2. DriveApp access by region or team.
3. Project run and edit permissions.
4. Catalog item and category visibility.

## Group security settings

In **Stage 1 > Security Settings**:

1. Create users and teams.
2. Assign users to teams.
3. Set Project-level permissions (typically **Run** only for non-admin teams).

## DriveApp permissions

In **Stage 7 DriveApp**:

1. Open the **Admin** dashboard.
2. Add or remove Project items in the catalog.
3. Configure category content visibility for teams.
4. Validate the tab-level settings (General, Catalog, Custom Fields, Orders, Tabs, Columns).

## Regional access

- Map teams to regions explicitly.
- Multi-region user setups can cause runtime ambiguity when configured incorrectly.
- Revalidate permissions after a Group copy or environment restore.

## Recover from permission drift

If every category becomes unavailable:

1. Compare the security teams with a known-good Group.
2. Recreate the missing administrator team structure if needed.
3. Copy security users and teams from a working environment.
