## Push DEV changes to LIVE

Before copying the Group:

1. Reset form defaults in all changed Projects.
2. Validate teams and user permissions.
3. Save a backup of the current LIVE Group.
4. Close open Projects in DriveWorks Administrator.

Then follow the Group copy procedure in [Data Management](/driveworks/how-to/data-management#copy-group-dev-to-live).

## Form and Rule maintenance

### Form control image size

Use image assets near 2× the intended display size to balance quality and performance.

### Form control return values

1. Create a display table (for example `XXXTableDisplay`).
2. Export document data to that table.
3. Bind form controls to the returned values.

### Add a new data field

To add a field such as an address line:

1. Add the form control in the Project.
2. Add a matching column in the calculation table.
3. Add the matching column in SQL with an appropriate data type.

## Group table updates

For pricing or material updates:

1. Update the region spreadsheet source.
2. Refresh the macro-generated table output.
3. Copy final values into Group tables in Stage 1.

## Project-level macro reuse

When copying macro behaviour between Projects:

1. Open the source and target Projects in separate sessions.
2. Copy the required macros, then the related Variables and Constants.
3. Recreate the required Specification Flow operations and transitions.
4. Test in DriveWorks Live and validate SQL outcomes.

## Quality and diagnostics

### Throw an error on colour TBC

Use a boolean or table flag to disable draft generation controls and notify users that unresolved colour selections remain.

### Change unit system

Generation Tasks can change drawing unit systems. Validate positional properties against the unit system in effect at task execution time.

### Verbose Specification Explorer logging

Enable Project logging through Project settings when you need deeper diagnostics.
