Move Groups and Projects between environments with the **DriveWorks Data Management** tool. Copy Group promotes DEV to LIVE or creates a backup Group; Pack and Go produces a portable package. Back up SQL and the target Group before any copy.

## Copy Group (DEV to LIVE)

Before you start:

1. Back up the SQL databases.
2. Confirm the target Group data is backed up.
3. Close open DriveWorks Administrator Projects.

Then:

1. Open **DriveWorks Data Management**.
2. Log in to the source Group.
3. Open **DriveWorks Tasks > Copy Group**.
4. Browse to the target Group and connect.
5. Select the required Projects and rule history.
6. Select the required Group tables.
7. Select **Copy new security users and teams** and **Copy all connectors** where needed.
8. Review the included files and models.
9. Finish and validate the copied output.

## Copy Group (create a new backup Group)

1. In Copy Group, choose **Create new group**.
2. Choose **Shared**.
3. Provide the Group name and server.
4. Complete the connection and copy wizard.
5. Exclude the **Quotes** folder unless the Specifications are intentionally required.

## Copy a Project within a Group

1. Close the Project in all sessions.
2. Prepare the target folder.
3. Use **Import Project**.
4. Validate the model references.
5. Re-apply Project visibility and DriveApp catalog permissions.

## Copy a Project to a different Group

1. Use the **Copy Group** wizard with project-scoped include selection.
2. Include only the required Project and model folders.
3. Validate in the target Group and update permissions.

## Pack and Go (package a Group)

Use **DriveWorks Tasks > Pack and Go** to create a package:

1. Select Projects and rule history.
2. Select the required tables and components.
3. Include connectors as required.
4. Avoid unnecessary extra content if packaging fails with date-parsing errors.

Pack and Go produces an Individual Group package, not a Shared Group runtime deployment.

## After a copy

### Relink Group settings

1. Open the copied Group in DriveWorks Administrator.
2. Update the specification and root folder settings.
3. Relink DriveApps and validate the CPQ database bindings.

### Update Group tables

Update the Group table database names (for example `DEVAutexNZCPQ`) after an environment copy.

### Start Autopilot

Open **DriveWorks Autopilot** and start the required Groups after copy operations.
