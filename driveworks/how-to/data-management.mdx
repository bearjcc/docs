## Copy Group (DEV to LIVE)

### Preconditions

1. Backup SQL databases.
2. Confirm target Group data is backed up.
3. Close open DriveWorks Administrator projects.

### Procedure

1. Open DriveWorks Data Management.
2. Log in to source Group.
3. Open `DriveWorks Tasks > Copy Group`.
4. Browse target Group and connect successfully.
5. Select required projects and rule history.
6. Select required Group tables.
7. Select `Copy new security users and teams` and `Copy all connectors` where needed.
8. Review included files and models.
9. Finish and validate copied output.

## Copy Group (Create New Backup Group)

1. In Copy Group, choose `Create new group`.
2. Choose `Shared`.
3. Provide Group name and server.
4. Complete connection and copy wizard.
5. Exclude `Quotes` folder unless specifications are intentionally required.

## Copy Project Within a Group

1. Close project in all sessions.
2. Prepare target folder.
3. Use `Import Project`.
4. Validate model references.
5. Re-apply project visibility and DriveApp catalog permissions.

## Copy Project to a Different Group

1. Use `Copy Group` wizard with project-scoped include selection.
2. Include only required project and model folders.
3. Validate in target Group and update permissions.

## Pack and Go (Package Group)

Use `DriveWorks Tasks > Pack and Go` to create a package:

1. Select projects and rule history.
2. Select required tables and components.
3. Include connectors as required.
4. Avoid unnecessary extra content if packaging fails with date parsing issues.

Pack and Go produces an Individual Group package, not a Shared Group runtime deployment.

## Post-Copy Tasks

### Relink Group Settings

1. Open copied Group in DriveWorks Administrator.
2. Update specification/root folder settings.
3. Relink DriveApps and validate CPQ database bindings.

### Update Group Tables

Update Group table database names (for example `DEVAutexNZCPQ`) after environment copy.

### Start Autopilot

Open DriveWorks Autopilot and start required Groups after copy operations.
