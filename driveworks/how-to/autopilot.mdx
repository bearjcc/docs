When the model generation queue stalls or the SOLIDWORKS integration blocks, restart Autopilot — first the agent, then the host machine if that does not clear it.

## Stop and start Autopilot

Use this first when the queue stalls but Autopilot is still responsive.

<Steps>
## Stop the agent

Click the **Stop** button and wait for the stopped state to appear in the log.

## Start the agent

Click the **Start** button and confirm the model queue resumes.

## Restart SOLIDWORKS if needed

If generation is still blocked, restart SOLIDWORKS on the Autopilot host, then start the agent again.
</Steps>

## Restart the Autopilot host

Use when Autopilot controls are unavailable or every queue is stalled.

<Steps>
## Connect to the host

Remote desktop to `azdwapp01.autexnz.local` and restart Windows.

## Reopen Autopilot

Log in again and open **DriveWorks Autopilot**.

## Start both agents

Start `AutexGLOBAL`. Open a second Autopilot session and start `DEVAutexGLOBAL`.

<Check>Both queues resume from their prior state.</Check>
</Steps>
