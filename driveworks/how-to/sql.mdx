SQL Server backs the DriveWorks Groups and CPQ data. Use SQL Server Management Studio (SSMS) to query and maintain tables, back up and restore databases, and recover items stuck after a crash. Back up before any destructive change.

## Run SQL queries

1. Open **SQL Server Management Studio**.
2. Open the target table with **Select Top 1000 Rows**, or open a new query window.
3. Write and execute the statement.

```sql
SELECT * FROM dbo.Specifications;
```

## Create and link a table

### Create the table

1. Create the table in the SSMS UI.
2. Use a unique ID column with a `newid()` default where appropriate.
3. Add the required data columns.

### Link it to DriveWorks

1. In DriveWorks Administrator, create a **Stage 4 > Documents > Data** export.
2. Choose the SQL Server data export.
3. Configure the server, authentication, database, and table.
4. Map DriveWorks values to the SQL columns.
5. Trigger the export from Project macros.

## Edit and delete tables

- Edit columns in the SQL table design view.
- Refresh query views (<kbd>Ctrl</kbd> <kbd>Shift</kbd> <kbd>R</kbd>) after schema changes.
- Before deleting a table, check dependencies and remove the linked DriveWorks logic first.

## Back up and restore a database

### Back up manually

1. Create a backup folder.
2. In SSMS, open the database **Tasks > Back Up**.
3. Replace the destination path and run the backup.
4. Repeat for all required databases.

### Restore manually

1. In SSMS, select **Databases > Restore Database**.
2. Select the backup device file.
3. Provide the target database name.
4. Use the overwrite option where needed.

## Delete database contents

<Warning>
Destructive and irreversible. Only for controlled environment resets. Back up first, confirm the target database name carefully, and run cleanup scripts one step at a time.
</Warning>

## Unlock a frozen item

When a DriveApp item is stuck in the `running` state after a crash:

1. Find the `OrderNumber` in `driveApp-CPQ-Order`.
2. Resolve the related `SpecificationId` via `driveApp-CPQ-OrderItems`.
3. Update the state values in `AutexGLOBAL.dbo.Specifications` to the unlocked state.

```sql
UPDATE [AutexGLOBAL].[dbo].[Specifications]
SET StateId = '<unlocked-state-id>', StateName = 'Complete', StateType = 1
WHERE Id = '<specification-id>';
```
