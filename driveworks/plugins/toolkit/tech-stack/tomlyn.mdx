**Tomlyn** is the .NET TOML library behind Toolkit snapshot export, import, and **Group Constants** storage. It turns in-memory `ProjectData` / `GroupData` models into human-readable files that diff cleanly in Git and review tools.

## What it is

[Tomlyn](https://github.com/xoofx/Tomlyn) is a TOML parser and serializer for .NET. Toolkit builds `Tomlyn.Model` tables (`TomlTable`, arrays, multiline strings) and round-trips them with `Toml.Parse` / `Toml.FromModel` / `Toml.ToModel`.

The package is a compile-time dependency embedded in **`Autex Toolkit.dll`** (via Costura.Fody), same as Serilog and Scriban.

## Usage in Autex Toolkit

Tomlyn usage lives under `Autex Toolkit/DWPlugin/Core/`. Search `DWPlugin` for `using Tomlyn` to find all call sites.

### Group Constants

`GroupConstantsTomlIO` reads and writes `GroupContentFolder\AutexToolkit\GroupConstants.toml` via Tomlyn, aligned with snapshot TOML style (multiline `Value` / `Comment` keys). See [Group Constants](/driveworks/plugins/toolkit/group-constants).

### Snapshot export and import

| Type | Role |
| --- | --- |
| `SnapshotTomlSerializer` | `ProjectData` / `GroupData` → TOML; projected sub-tables, omitted defaults, sorted export |
| `SnapshotTomlDeserializer` | TOML → models; restores projected collections and specification flow |
| `SnapshotTomlDeltaExportHelper` | Baseline capture and companion `*.delta.toml` on incremental export |
| `SnapshotTomlSectionCodecs` | Section-level encode/decode helpers shared by serializer and tests |
| `SpecificationFlowTomlMapper` | Nested TOML for specification flow definitions |

Release scripts **`ExportGroupProjectToml`** and Version Control **`ExportToGitFolderToml`** write snapshot TOML to disk; delta mode uses `SnapshotTomlDeltaExportHelper`.

### Project compare and version control

Comparison sources resolve **TOML files** and **TOML at a Git ref** (`TomlAtGitRefSource`). `VcTomlReportArtifactManifest` persists VC report metadata as TOML. Compare workflows consume the same snapshot shape as export.

## Why we picked it

- **Human-readable snapshots:** TOML sections and keys read like config, not opaque blobs.
- **Diff-friendly exports:** Stable sort order, sparse defaults, and delta files make Git and code review practical for **Project** and **Group** changes.
- **TOML over JSON:** Line-oriented structure and comment support produce cleaner diffs than JSON for large project exports (JSON remains for legacy `GroupConstants.json` and WebView bridge payloads).
