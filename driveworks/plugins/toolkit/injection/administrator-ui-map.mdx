This page names the regions of **DriveWorks Administrator** for maintainer work: what users see, what the decompiled WPF tree calls each area, and which injection surface to use when extending the product. Code identifiers below come from decompiled **23.1.0.630** (`DriveWorks.Applications.Implementation`, `DriveWorksAdministratorCore`, `DriveWorks.Applications.AdministratorViews`). Names are stable across recent releases; always confirm against the Live Visual Tree when debugging a specific build.

Use this map with the [injection guides](/driveworks/plugins/toolkit/injection) when choosing siblings, parents, and APIs.

## Terminology bridge

DriveWorks docs and Toolkit guides use product names; decompiled XAML and reflection use different labels for the same real estate.

| User / Wiki term         | Decompiled control or type                                | SDK or service                                                                                              |
| ------------------------ | --------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------- |
| QuickBar                 | `QuickBar` (`DriveWorks.Applications.Support.CommandBar`) | `IQuickBarManager`                                                                                          |
| Command Manager          | `CommandBar`                                              | Application `ICommandBarManager`; per-view `IViewCommandBarManager` on `IViewEnvironment.CommandBarManager` |
| Command group            | `ICommandBarGroup`                                        | `AddGroup(...)` then `AddCommandButton(...)`                                                                |
| Commands registry        | `CommandManager` (implementation type)                    | `ICommandManager` (`RegisterCommand`, `InvokeCommand`)                                                      |
| Left sidebar / ToDo tree | `ToDo` (`TreeView`), `ViewTreeViewModel`                  | `IViewManager`, `IViewRegistrationService` / `InsertControlViewAfter`                                       |
| Content area             | `ContentHost` (`DockPanel`)                               | Hosts active `IView`; WinForms views use `DriveWorksWindowsFormsHost`                                       |
| Status bar               | `StatusBar`, `StatusBarItem` children                     | `IStatusBarService`                                                                                         |
| Tree (in a view)         | `CommandTreeView`                                         | Selection, drag-drop, filter via `FilterTextBox`                                                            |
| Property grid            | `PropertyGrid`, `FormDesignerPropertyGrid`, hosts         | `DynamicProperty`, custom `IPropertyEditor`                                                                 |

<Info>
  **Command Manager** in Wiki and Toolkit docs is the **`CommandBar` WPF
  control**, not `ICommandManager`. Commands are registered on
  `ICommandManager`; buttons are placed on `CommandBar` groups.
</Info>

## Application shell (`SolidWorksWindow`)

The Administrator main window is `SolidWorksWindow` (factory: `SolidWorksChromeFactory`), XAML `solidworkswindow.xaml`. `IComponentConnector.Connect` cases 1-25 name the shell regions.

Editable source: [`administrator-ui-shell.drawio`](/images/wiki/driveworks/administrator-ui-shell.drawio).

### Shell region reference

| Friendly name      | Code name                         | WPF type / notes                                                                                                                   |
| ------------------ | --------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| Application window | `MainWindow` / `SolidWorksWindow` | Root `Window`                                                                                                                      |
| Title bar chrome   | `TitleCommandBarBacking`          | `CommandBar`; drag and double-click maximize                                                                                       |
| App icon           | `ApplicationIconButton`           | `Button`                                                                                                                           |
| App title label    | `ApplicationName`                 | `Label`                                                                                                                            |
| QuickBar           | `QuickBar`                        | Populated by `State_UpdateQuickBar` / `IQuickBarManager`                                                                           |
| Window buttons     | `WindowsButtonsBar`               | `HelpButton`, `MinimizeButton`, `MaxRestoreButton`, `CloseButton`                                                                  |
| Command Manager    | `CommandBar`                      | Merges `ICommandBarManager` groups with active view `IViewCommandBarManager` groups (`State_UpdateCommandBar`)                     |
| Body grid          | `ApplicationBodyGrid`             | `Grid`                                                                                                                             |
| Sidebar column     | `TodoColumnDefinition`            | `ColumnDefinition` width persistence                                                                                               |
| ToDo tree          | `ToDo`                            | `TreeView` bound to `ViewTreeViewModel.RootViews`                                                                                  |
| Sidebar splitter   | `TodoSplitter`                    | `GridSplitter`                                                                                                                     |
| Content host       | `ContentHost`                     | `DockPanel`; children cleared on `ActiveViewChanged`                                                                               |
| Status bar         | `StatusBar`                       | `ReadyStateText`, `VersionState`, `BuildStatus`, `LicenseTypeStatus`, `GroupStatus`, `ProjectStatus`, `UserStatus`, `ServerStatus` |

When the active view changes, `mViewManager_ActiveViewChanged` clears `ContentHost`, mounts the new `IView`, updates `mActiveViewCommandBarManager`, and rebuilds the Command Manager bar.

## ToDo tree (views and siblings)

Sidebar items are not arbitrary WPF nodes. Each entry is a `ViewDefinition` (`Name`, `Title`, `Parent`) wrapped in `ViewDefinitionViewModel` with optional `Children`. Clicking the tree calls `IViewManager.ActivateView`.

Constants: `DriveWorks.Applications.Administrator.Extensibility.AdministratorViewNames`.

Injection uses `parentViewName` (stage) and `afterViewName` (sibling order). See [Adding a Left Sidebar Menu Item](/driveworks/plugins/toolkit/injection/add-sidebar-menu-item).

## Active view patterns (`ContentHost`)

Views hosted in `ContentHost` implement `IView`. Layout varies; three recurring shapes matter for injection.

| Archetype           | Example type                                  | Notable regions                                                                                                              | Injection                                                                                                                                                                                                     |
| ------------------- | --------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Command bar + trees | `MacroFlowsViewControl`                       | `MacrosTree`, `ToolboxTree`, `ConditionsToolboxTree`, `CategoryFilter`, `ToolboxFilter`, `MacrosSplitter`, `ToolboxSplitter` | [View-scoped command button](/driveworks/plugins/toolkit/injection/add-view-scoped-command-button)                                                                                                            |
| Web designer        | `FormDesignerView`                            | `ControlTreeView`, `WebFormDesigner`, `DesignerPropertyGrid`, `DesignerContextMenu`                                          | [Form Designer context menu](/driveworks/plugins/toolkit/injection/add-form-designer-context-menu-item), [property grid](/driveworks/plugins/toolkit/injection/add-form-control-properties), view command bar |
| List / grid views   | `VariablesViewControl`, document admins, etc. | Often `CommandTreeView` or list + detail pane                                                                                | View-scoped commands when `IViewEnvironment.CommandBarManager` is present                                                                                                                                     |

`IViewEnvironment` exposes `ViewName`, `CommandManager`, and `CommandBarManager` for the active view. Form Designer may omit `CommandBarManager` while still exposing the property grid and designer context menu.
