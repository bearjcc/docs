## Suggested reading order

Follow this tour when learning the codebase top-down.

### 1. Plugin Entry Point

Start here. The AutexToolkitPlugin class bootstraps the entire plugin when DriveWorks Administrator loads it. It registers the Autex Tools window, wires up the presenter, and initializes the Rules Builder integration.

Primary nodes:

- `Autex Toolkit/DWPlugin/Plugin/ToolkitPlugin.cs`

### 2. Presenter & Main Window

The AutexToolsPresenter orchestrates the plugin's UI lifecycle, connecting the DriveWorks SDK interface to the WPF main window. It manages view state transitions and dispatches user actions to scripts and services.

Primary nodes:

- `Autex Toolkit/DWPlugin/AutexToolsPresenter.cs`
- `Autex Toolkit/DWPlugin/Views/AutexToolsWindow.xaml`
- `Autex Toolkit/DWPlugin/Views/AutexToolsWindow.xaml.cs`

### 3. DriveWorks SDK Interface

DriveWorksInterface is the gateway to the DriveWorks SDK. It wraps all SDK calls for project/group data access, rule analysis, form control manipulation, and document generation. Script helpers extend it with domain-specific operations.

Primary nodes:

- `Autex Toolkit/DWPlugin/Infrastructure/DriveWorks/DriveWorksInterface.cs`
- `Autex Toolkit/DWPlugin/Infrastructure/DriveWorks/ScriptHelpers/ScriptHelpers.cs`
- `Autex Toolkit/DWPlugin/Infrastructure/DriveWorks/ScriptHelpers/ScriptHelpers.RoleComposition.cs`

### 4. Data Models

The unified data model centers on ProjectData and GroupData, which represent snapshots of DriveWorks projects and groups. These models support comparison, TOML serialization, and snapshot delta operations.

Primary nodes:

- `Autex Toolkit/DWPlugin/Models/GroupData.cs`
- `Autex Toolkit/DWPlugin/Models/ProjectData.cs`

### 5. Comparison Engine

The comparison engine compares two ProjectData snapshots, producing detailed change trees with typed comparers for constants, variables, forms, and settings. It powers the Project Compare feature and parity validation.

Primary nodes:

- `Autex Toolkit/DWPlugin/Core/Comparison/ChangeTreeBuilder.cs`
- `Autex Toolkit/DWPlugin/Core/Comparison/ComparisonSession.cs`
- `Autex Toolkit/DWPlugin/Core/Comparison/KnownLimitationPolicy.cs`
- `Autex Toolkit/DWPlugin/Core/Comparison/ProjectDataComparisonEngine.cs`

### 6. Snapshot & TOML Serialization

TOML serialization and deserialization converts between ProjectData models and human-readable TOML files for git-based version control. The snapshot delta system tracks changes between revisions.

Primary nodes:

- `Autex Toolkit/DWPlugin/Core/SnapshotTomlDeltaExportHelper.cs`
- `Autex Toolkit/DWPlugin/Core/SnapshotTomlDeserializer.cs`
- `Autex Toolkit/DWPlugin/Core/SnapshotTomlSerializer.cs`

### 7. Script System

The script system provides a plugin-within-a-plugin architecture. Scripts extend AutexScriptContext to perform operations like data export, bulk refactors, validation, and diagnostics. Scripts are categorized as Release, Dev, Test, WIP, or Version Control.

Primary nodes:

- `Autex Toolkit/DWPlugin/Core/Scripts/AutexScript.cs`
- `Autex Toolkit/DWPlugin/Core/Scripts/AutexScriptContext.cs`
- `Autex Toolkit/DWPlugin/Core/Scripts/AutexScriptRegistry.cs`
- `Autex Toolkit/DWPlugin/Core/Scripts/BaseAutexScript.cs`
- `Autex Toolkit/DWPlugin/Core/Scripts/IAutexScript.cs`

### 8. WebView UI Layer

The WebView2-based UI renders HTML/CSS/JS surfaces for Script Tools, Project Comparison, Version Control, and various pickers. Page builders compose Scriban templates with shared CSS/JS components.

Primary nodes:

- `Autex Toolkit/DWPlugin/WebAssets/ProjectComparison/project-comparison.app.js`
- `Autex Toolkit/DWPlugin/WebAssets/ScriptTools/script-tools.app.js`
- `Autex Toolkit/DWPlugin/WebView/ProjectComparisonWebViewPageBuilder.cs`
- `Autex Toolkit/DWPlugin/WebView/ScriptToolsWebViewPageBuilder.cs`
- `Autex Toolkit/DWPlugin/WebView/ToolkitWebViewHostControl.cs`
- `Autex Toolkit/DWPlugin/WebView/WebView2BridgeV1.cs`

### 9. Rules Builder Extensions

Rules Builder integration adds spell checking, icon picking, metaclass picking, group constant insertion, and rule formatting directly into the DriveWorks Rules Builder editor.

Primary nodes:

- `Autex Toolkit/DWPlugin/RulesBuilder/AutexIconPickerWindow.cs`
- `Autex Toolkit/DWPlugin/RulesBuilder/MetaclassWebPickerWindow.cs`
- `Autex Toolkit/DWPlugin/RulesBuilder/RulesBuilderButtonInjection.cs`
- `Autex Toolkit/DWPlugin/RulesBuilder/RulesBuilderSpellCheckEntryPoint.cs`

### 10. Build & Test Infrastructure

PowerShell build scripts compile the plugin, run tests, and produce release artifacts. The RuleFormatter.Tests project validates core formatting and cross-project connection logic.

Primary nodes:

- `Autex Toolkit/Build.ps1`
- `Autex Toolkit/Build-Release.ps1`
- `Autex Toolkit/RuleFormatter.Tests/GroupConnectionIndexTests.cs`
- `Autex Toolkit/RuleFormatter.Tests/RuleFormatterTests.cs`
- `Autex Toolkit/version.json`
