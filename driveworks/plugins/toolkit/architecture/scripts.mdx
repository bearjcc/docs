Concrete **Autex Toolkit** script implementations — the operations users pick in **Autex Tools**. Folders encode maturity and support expectations.

## In this layer

| Folder | Purpose |
| --- | --- |
| `Scripts/Release/` | Production scripts shipped to users; documented in the [Release scripts catalog](/driveworks/plugins/toolkit/scripts-release-catalog) |
| `Scripts/Dev/` | Exploratory probes and one-off diagnostics |
| `Scripts/Test/` | Validation and parity harnesses |
| `Scripts/WIP/` | In-progress work not ready for Release |
| `Scripts/VersionControl/` | Git/TOML export, import, and pre-push validation |

Boundary: script base types, registry, and context live in [Core Services](/driveworks/plugins/toolkit/architecture/core-services); the Autex Tools UI that lists and runs scripts spans [WPF Views](/driveworks/plugins/toolkit/architecture/wpf-views) and [WebView UI](/driveworks/plugins/toolkit/architecture/webview-ui).
