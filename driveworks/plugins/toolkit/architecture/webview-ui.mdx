**WebView UI** is the HTML/CSS/JS half of Toolkit surfaces hosted in **WebView2**. C# page builders compose Scriban templates; front-end apps talk to the host through a shared **bridge** (`bridge.js`).

## In this layer

- **WebAssets** — per-feature folders (`ProjectComparison/`, `ScriptTools/`, `VersionControl/`, pickers) plus `_shared/` partials and JS utilities.
- **WebView** — `*WebViewPageBuilder.cs`, `ToolkitWebViewHostControl`, `WebView2BridgeV1` and bridge action handlers.
- **Template composition** — Scriban partials for trees, filters, workflow strips, and picker panels.

Boundary: WPF shells that embed WebView controls live in [WPF Views](/driveworks/plugins/toolkit/architecture/wpf-views); Rules Builder picker windows bridge from [Rules Builder Integration](/driveworks/plugins/toolkit/architecture/rules-builder).
