**Build & Infrastructure** spans repo-root tooling: **`Build.ps1`**, release deploy, test projects, Understand-Anything graph artifacts, and maintainer docs. Agents and contributors should use these entry points rather than raw MSBuild on the plugin `.csproj`.

## In this layer

- **Build scripts** — `Build.ps1`, `Build-Release.ps1`, debug helpers; version bump via `version.json`.
- **Project config** — `Autex Toolkit.csproj`, Fody weavers, solution layout.
- **Tests** — `RuleFormatter.Tests` and other validation assemblies.
- **Analysis artifacts** — `.understand-anything/` intermediate files feeding architecture docs and `export-architecture-layers.py`.
- **Docs** — `README.md`, `AGENTS.md`, `docs/SCRIPTS.md` at Toolkit root.

Boundary: plugin runtime code lives in other layers; this layer is how you compile, release, and regenerate architecture metadata.
