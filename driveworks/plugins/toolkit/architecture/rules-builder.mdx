**Rules Builder Integration** extends the DriveWorks **Rule Builder** editor: toolbar buttons, spell checking, pickers, formatting, and **Form Designer** property-grid hooks. Most pickers reuse [WebView UI](/driveworks/plugins/toolkit/architecture/webview-ui) assets inside WPF host windows.

## In this layer

- **Button injection** — `RulesBuilderButtonInjection` registers Autex commands on the Rule Builder toolbar.
- **Entry points** — one class per feature (spell check, icon picker, metaclass picker, group constants, formatter, rule analysis).
- **Picker windows** — `AutexIconPickerWindow`, `MetaclassWebPickerWindow`, `GroupConstantWebPickerWindow`.
- **Form Designer** — layout grid bridge and dynamic property registration under `FormDesigner/`.

Boundary: shared spelling and icon assets live in [Assets & Services](/driveworks/plugins/toolkit/architecture/assets-and-services); generic Administrator injection patterns are documented under [Injection guides](/driveworks/plugins/toolkit/injection).
