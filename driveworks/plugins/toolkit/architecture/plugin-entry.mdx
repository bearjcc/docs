Start here when tracing how **Autex Toolkit** loads inside **DriveWorks Administrator**. This layer covers plugin registration, the **Autex Tools** presenter, and the **DriveWorks SDK** boundary scripts call through.

## In this layer

- **Plugin bootstrap** — `AutexToolkitPlugin` registers commands, context menus, and view lifecycle hooks.
- **Presenter** — `AutexToolsPresenter` wires the script registry to run, abort, export, and cache refresh.
- **SDK gateway** — `DriveWorksInterface` and `ScriptHelpers` centralize Group/Project access for scripts and UI.

Boundary: host integration and shared helpers live here; comparison logic, data models, and script implementations live in other layers.
