**WPF Views** are native **DriveWorks Administrator** windows and pages: the **Autex Tools** main window, script browser/runner pages, version-control shell, and auxiliary dialogs. Most host [WebView UI](/driveworks/plugins/toolkit/architecture/webview-ui) surfaces inside XAML layouts.

## In this layer

- **Main window** — `AutexToolsWindow.xaml` and code-behind; entry chrome for Toolkit in Administrator.
- **Script UX** — `ScriptBrowserPage`, `ScriptRunnerPage`, `ScriptShellView` (WebView bridge partial).
- **Version control** — `VersionControlWindow`, `VcShellView`.
- **Utilities** — `GroupConstantsWindow`, `InputBoxWindow`, overlay buttons.

Boundary: plugin registration and presenter logic sit in [Plugin Entry](/driveworks/plugins/toolkit/architecture/plugin-entry); HTML/JS assets sit in WebView UI.
