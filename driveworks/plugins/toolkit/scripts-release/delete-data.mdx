## Delete data

This Release script removes **one** item per run, chosen by **Data type** and either **Name** or **Form Messages** numeric **Code**. It uses the same delete paths as **Unreferenced data** and ships with **Writes on Run** behavior when **Confirm delete** is checked.

## How to open and run

1. Open **DriveWorks Administrator**, connect to the **Group** (and open a **Project** when deleting **Project**-scoped types).
2. Open **Autex Tools** ( **Autex Toolkit** / **DW Tools** ) and pick **Delete data**.
3. Choose **Data type** from the dropdown (`ScriptDataTypeLists.ForDeleteData` in source).
4. Enter **Name** (or **Code** when **Form Messages** is selected).
5. Check **Confirm delete** (required; without it the script stops with a blocked result).
6. Choose **Run** once. Successful **Project** deletes call `Project.Save()` in the script.

For **Group Tables**, **Teams**, or **Users**, an open **Group** is required; **Name** identifies the row. Other types require an open **Project** (the active **Project** in **Project** mode). In **Group** mode, **Project**-level types follow `ShouldLoopOverProjectsWhenGroupMode` (non-group-only types loop per **Project**).

## Prerequisites

- Correct **Group**/**Project** context for the selected **Data type**.
- Valid identifier: non-empty **Name** except **Form Messages**, which requires a non-zero integer **Code** (defaults to `1001` in inputs until you change it).

## Safety

This script sets `MutationsOnlyInApplySelected` to **false**: **Run performs the delete** once **Confirm delete** is true.

<Warning>
  Deleted **Project** data is persisted immediately (`Save()` on success). Treat
  deletes as irreversible without backup or snapshots.
</Warning>

## Verify labels in build

Confirm **Delete data**, **Data type**, **Name**, **Code**, **Confirm delete**, and blocked message text in **Autex Tools** align with `DeleteDataScript.BuildInputSpecs()` and `Run()`.
