**AutexCustom** ships as `AutexCustom.dll` (Shared Project Extender plus document designers). Deploy with **AutexPdfExport** when Projects use the PDF document type; deploy with **Autex Toolkit** when using form-control conversion tasks.

<Steps>
## Debug build

```powershell
.\AutexCustom\Build.ps1
```

Output: `AutexCustom\bin\Debug\AutexCustom.dll` (and copied PdfExport satellites when the project references build).

## Release deploy

From `main`:

```powershell
.\AutexCustom\Build-Release.ps1
```

## Copy to plugin folder

Place `AutexCustom.dll` (and `AutexPdfExport.dll` when needed) in the DriveWorks plugins folder for your environment. Restart DriveWorks so extenders register.

## Register in Project

Enable the extender in the target **Project** per your Autex deployment checklist. Confirm **Functions** categories and document designers appear in Administrator.

<Check>`AutexCustomPluginLoaded()` returns true in a test Rule.</Check>
</Steps>
