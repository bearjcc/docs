## Goals

1. Package released Group states as semantic versions (`1.0.0`, `1.0.1`, and so on).
2. Persist both Group version and product version for line-item traceability.
3. Record bug fixes and features included in each release.

## Version Stamping (Specification Flow)

Use a macro on the `Specify` state entry for new specifications:

1. Open Specification Flow and edit the `Specify` state.
2. Add an Enter State macro (for example `UpdateVersion`).
3. Add a condition for specification type `New` (not `NewCopiedExisting`).
4. Write version constants once and do not overwrite them.

## Implemented Process

Use `S:\DRIVEWORKS BACKUPS\` with two folders:

- `_Archived` for backup packages.
- `_Released` for synchronized DEV-to-LIVE release packages only.

Each release folder (for example `1.0.0`) should include:

1. DriveWorks package file.
2. Bug fix and feature list.
3. Test documents (draft and released quote/job sheet).
4. Database backup set (for example `AutexGLOBAL`, `AutexNZCPQ`, `AutexNZQuoteInfo`).
